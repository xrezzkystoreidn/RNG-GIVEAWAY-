🤖 Bot Integration Docs

Perintah Firestore yang perlu disambungkan dari bot ke Firebase project wa-bot-fa92c.

📁 Struktur Koleksi Firestore

// Collections yang dipakai panel ini: users/ → data semua user (role, saldo, nama, phone, dll) orders/ → semua transaksi order dari user deposits/ → semua request deposit dari user config/site → setting toko (nama, WA, payment, dll) config/saldootp → saldo & config OTP provider otp_logs/ → log setiap OTP yang terkirim otp_topup_log/ → log top-up saldo OTP oleh admin

👤 1. User Daftar via Bot (/daftar)

Simpan ke users/{phoneNumber} — role WAJIB 'user', bukan 'admin':

// Node.js / Firebase Admin SDK await db.collection('users').doc(phoneNumber).set({ uid: phoneNumber, phone: phoneNumber, // format: 6281234567890 name: namaUser, pushName: waPushName, // nama di WA email: null, role: 'user', // ← SELALU 'user', jangan 'admin'! saldo: 0, totalDeposit: 0, totalOrder: 0, isActive: true, isBanned: false, registeredFrom: 'whatsapp-bot', createdAt: admin.firestore.FieldValue.serverTimestamp(), lastActive: admin.firestore.FieldValue.serverTimestamp() });

💰 2. User Request Deposit (/deposit)

// Buat dokumen baru di deposits/, status 'pending' await db.collection('deposits').add({ userId: phoneNumber, phone: phoneNumber, name: namaUser, amount: nominalDeposit, // integer (Rp) method: 'Transfer Bank', refCode: generateRefCode(), // kode unik opsional note: catatanUser, status: 'pending', // admin panel yang approve createdAt: admin.firestore.FieldValue.serverTimestamp() }); // Admin approve → saldo user otomatis bertambah (handled panel)

📦 3. User Order (/order atau checkout)

// 1. Cek saldo dulu const userDoc = await db.collection('users').doc(phoneNumber).get(); const { saldo, isBanned } = userDoc.data(); if (isBanned) return bot.reply('❌ Akun kamu dibanned.'); if (saldo < hargaProduk) return bot.reply('❌ Saldo tidak cukup.'); // 2. Buat order + kurangi saldo (pakai batch biar atomik) const batch = db.batch(); const orderRef = db.collection('orders').doc(); batch.set(orderRef, { userId: phoneNumber, phone: phoneNumber, service: namaLayanan, // 'Mobile Legends 86 Diamond' layanan: namaLayanan, kategori: kategoriProduk, // 'Mobile Legends' target: targetId, // ID game / nomor tujuan harga: hargaProduk, price: hargaProduk, catatan: catatanUser, status: 'pending', // admin panel yang proses externalId: null, createdAt: admin.firestore.FieldValue.serverTimestamp() }); batch.update(db.collection('users').doc(phoneNumber), { saldo: admin.firestore.FieldValue.increment(-hargaProduk), totalOrder: admin.firestore.FieldValue.increment(1), lastActive: admin.firestore.FieldValue.serverTimestamp() }); await batch.commit(); bot.reply(`✅ Order dibuat! ID: ${orderRef.id.slice(-8)}`);

🔔 4. Notifikasi Perubahan Status Order (Realtime)

// Bot listen perubahan status order, kirim notif ke user: db.collection('orders').doc(orderId).onSnapshot(snap => { const { status, phone } = snap.data(); if (status === 'selesai') { bot.sendMessage(phone, `✅ Order kamu selesai diproses!`); } if (status === 'dibatalkan') { // saldo sudah dikembalikan otomatis oleh panel bot.sendMessage(phone, `❌ Order dibatalkan, saldo dikembalikan.`); } if (status === 'diproses') { bot.sendMessage(phone, `🔄 Order kamu sedang diproses admin.`); } });

⚙️ 5. Baca Config Toko (untuk menu bot)

// Baca sebelum proses request user: const cfg = (await db.collection('config').doc('site').get()).data(); if (cfg.maintenance) { return bot.reply(cfg.maintenanceMsg || '🔧 Sedang maintenance.'); } // Pakai cfg.waAdmin, cfg.payName, cfg.payNo, cfg.minDeposit, dll

🔑 6. Kirim OTP — Log & Kurangi Saldo

// Tiap kirim OTP, log dan kurangi saldo OTP: const otpCfg = (await db.collection('config').doc('saldootp').get()).data(); const biaya = otpCfg.ratePerOtp || 0; if (otpCfg.saldo < biaya) { console.error('Saldo OTP habis!'); return; } const batch = db.batch(); batch.set(db.collection('otp_logs').doc(), { phone: targetPhone, status: 'sent', // atau 'failed' cost: biaya, provider: otpCfg.provider, createdAt: admin.firestore.FieldValue.serverTimestamp() }); batch.update(db.collection('config').doc('saldootp'), { saldo: admin.firestore.FieldValue.increment(-biaya), totalUsed: admin.firestore.FieldValue.increment(biaya) }); await batch.commit();

🚦 7. Auto-Complete Order (jika punya API provider)

// Jika bot terintegrasi Digiflazz / Apigames / dll: // Setelah callback success dari provider: const batch = db.batch(); batch.update(db.collection('orders').doc(orderId), { status: 'selesai', externalId: providerTrxId, updatedAt: admin.firestore.FieldValue.serverTimestamp(), updatedBy: 'bot-auto' }); await batch.commit(); // Jika gagal/dibatalkan → refund saldo: const batch2 = db.batch(); batch2.update(db.collection('orders').doc(orderId), { status: 'dibatalkan', updatedAt: admin.firestore.FieldValue.serverTimestamp() }); batch2.update(db.collection('users').doc(phoneNumber), { saldo: admin.firestore.FieldValue.increment(+hargaProduk) }); await batch2.commit();

✅ Checklist Integrasi Bot

users/{phone} — set saat /daftar, role: 'user'

Panel pakai: tabel users, adjust saldo, role manager, history

deposits/{auto} — set saat /deposit, status: 'pending'

Panel pakai: approve → saldo user +amount otomatis

orders/{auto} — set saat checkout, status: 'pending'

Panel pakai: update status, cancel → refund saldo

config/site — baca untuk nama toko, WA, payment, maintenance

Di-update lewat panel Settings

config/saldootp — baca rate, kurangi saldo tiap kirim OTP

Di-top-up dari panel Saldo OTP

otp_logs/{auto} — log tiap OTP terkirim

Ditampilkan di panel tab Saldo OTP
